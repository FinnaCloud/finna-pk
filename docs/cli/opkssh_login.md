## finna-pk login

Authenticate with an OpenID Provider to generate an SSH key for finna-pk

### Synopsis

Login creates finna-pk SSH keys

Login generates a key pair, then opens a browser to authenticate the user with the OpenID Provider. Upon successful authentication, finna-pk creates an SSH public key (~/.ssh/id_ecdsa) containing the user's PK token. By default, this SSH key expires after 24 hours, after which the user must run "finna-pk login" again to generate a new key.

Users can then SSH into servers configured to use finna-pk as the AuthorizedKeysCommand. The server verifies the PK token and grants access if the token is valid and the user is authorized per the auth_id policy.
Arguments:
  alias      The provider alias to use. If not specified, the FINNA-PK_DEFAULT provider will be used. The aliases are defined by the FINNA-PK_PROVIDERS environment variable. The format is <alias>,<issuer>,<client_id>,<client_secret>,<scopes>


```
finna-pk login [alias] [flags]
```

### Examples

```
  finna-pk login
  finna-pk login google
  finna-pk login --provider=<issuer>,<client_id>,<client_secret>,<scopes>
```
